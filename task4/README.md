# Задание №4 (Удаление копий)

## Условие задачи
У вас есть папка, в которой хранятся резервные копии нескольких проектов. Каждый проект находится в отдельной папке, и каждая резервная копия представляет собой архивный файл с уникальным именем. Напишите сценарий оболочки, который удаляет все резервные копии для данного проекта, кроме последних 5.

---

### 1. Архитектура и особенности решения
Для реализации автоматической ротации архивов был разработан Bash-скрипт, осуществляющий сканирование всех подкаталогов в указанной директории по принципу DRY (Don't Repeat Yourself) и поддерживающий в них строго заданный лимит копий.

* **Автоматический обход всех проектов:** Скрипт принимает на вход путь к корневой директории бэкапов и самостоятельно перебирает все вложенные папки проектов за один проход.
* **Изоляция контекста:** Выполнение логики внутри каждой папки изолировано с помощью дочерних процессов. Это исключает поломку глобального цикла при сбоях команд перехода `cd`.
* **Защита имен с пробелами:** Использование системной утилиты `mapfile` гарантирует корректную обработку имен архивов, содержащих пробелы или спецсимволы.
* **Умная валидация состояния:** Внедрена опция `nullglob`, предотвращающая ложные срабатывания скрипта в пустых каталогах, а также реализован счетчик для логирования ситуаций, когда в корне бэкапов вообще нет проектов.

---

### 3. Алгоритм расследования и фильтрации файлов

Цель - отсортировать файлы по времени и отсечь ровно 5 самых новых. Логика построена на конвейере утилит:

#### **Шаг 1: Получение точного времени изменения**
Команда `find . -maxdepth 1 -type f -printf '%T@ %p\n'` опрашивает файловую систему и выводит только файлы верхнего уровня в виде: `<Unix_Timestamp> <путь_к_файлу>`. Это пресекает любые проблемы с разностью текстовых форматов дат в именах.

#### **Шаг 2: Сортировка по возрасту**
Команда `| sort -rn` выполняет обратную числовую сортировку. Наверх списка поднимаются самые свежие бэкапы.

#### **Шаг 3: Отсечение лимита хранения**
Команда `| tail -n +6` отбрасывает первые 5 строк (наиболее актуальные копии) и пропускает дальше по конвейеру только старые резервные копии, начиная с 6-й строки.

#### **Шаг 4: Нормализация путей**
Команда `| cut -d' ' -f2-` удаляет временную системную метку *Unix Timestamp*, возвращая конвейеру чистые имена файлов, которые затем считываются в системный массив.

### 4. Инструкция по использованию

#### **Шаг 1: Предоставление прав**
Перед запуском необходимо сделать файл сценария исполняемым на уровне операционной системы:
```
chmod +x cleanup_backups.sh
```

### *Шаг 2: Ручной запуск*
Скрипт вызывается с передачей одного параметра - целевого каталога хранения проектов:
```
./cleanup_backups.sh /home/app/backups/
```
